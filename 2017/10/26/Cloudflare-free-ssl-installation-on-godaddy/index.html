在Godaddy上安装Cloudflare免费SSL指南

发表于 2017-10-26 |


这篇文章的目的是快速完成设置过程。让我们建立了一个免费的（严格）SSL提供证书的CloudFlare，在一个Godaddy的共享的主机上。

本指南列出了要遵循的每个步骤，以使用免费的Cloudflare计划将您的网站访问者和CloudFlare AND之间的连接从CloudFlare加密到您的服务器。

为什么要使用SSL？

Cloudflare简要说明了这一点：“使用SSL来加密数据，如信用卡号码和其他敏感信息。SSL加密确保访问者和网站之间的通信保密”。

SSL的其他好处：

  • 访客有一种方法来验证他们在您的网站上，而不是冒名顶替者
  • 访问者可以验证您的网站的内容在运输过程中是否未被修改
  • 您网站的搜索引擎排名可能会有所改善

设置Cloudflare的CA证书

这里假设你已经把你的网站顺利的添加到Cloudflare，如果没有，请先完成网站的基本设置。

  • 转到“加密”页面
  • 点击“原始证书”框中的“创建证书”按钮

  • 选择“让Cloudflare生成私钥和CSR”
  • 输入您的域，然后点击下一步。请注意，您可以添加任何其他域，例如abc.com，def.com等

  • 保留默认的PEM密钥格式

  • 复制您的原始证书和私钥
  • 单击确定

有关创建证书的更多详细信息，可以参阅CloudFlare指南。
您现在应该全部设置，但是尚未将SSL转换为完整（严格）模式，因为我们需要首先在Godaddy上安装证书。

使用cPanel在Godaddy上安装SSL

这很容易，因为cPanel中有一个SSL部分，允许创建或上传证书和私钥。这也是我们安装证书的地方。

  • 在cPanel中，单击SSL / TLS

  • 转到“私钥（KEY）”页面，然后上传您之前生成的密钥

  • 上传“证书（CRT）”，和上面相同步骤
  • 点击“管理SSL站点”
  • 保留所有选项，并选择您的域
  • 应该出现一个新的“按域自动填充”按钮。点击它

  • 您的密钥和证书现在应该可见，只有最后一个输入应为空（CABUNDLE）。如果您尝试在此时安装原始CA，则可能会收到类似于此的错误：“ cloudflare ca bundle certificate authority bundle cloudflare the system did not find the certificate authority bundle that matches this certificate ”
  • 如果您收到错误，请输入以下提供的Cloudflare origin CA RS root(来源)
  • 点击“安装证书”按钮

CloudFlare Origin CA - RSA Root

最后一步！在Cloudflare上打开SSL

你现在应该准备好在您的网站上使用完整（严格）SSL。返回到Cloudflare上的CRYPTO页面，并选择“full（strict）”。

你应确保网站使用“https：//”网址，你可能需要添加重定向规则，并检查所有内容是否从安全网址加载。
